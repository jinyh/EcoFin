# EcoFin 项目配置

## 项目定位

金融经济百科知识库，采用中美对比视角，兼容 Obsidian 双向链接。

**核心特色**：
- 四学科体系：经济学、金融学、政治学、法学
- 中美对比视角：每个知识点同时呈现中美差异
- 五段式结构：理论 → 数学 → 对比 → 直觉 → 实践
- Obsidian 兼容：支持双向链接和知识图谱

**目标读者**：
- 投资者：理解市场机制、掌握资产配置
- 政策研究者：分析经济政策与制度框架
- 金融从业者：构建完整知识体系

## 文档格式规范

### YAML Frontmatter（必需）

每篇知识文档必须包含完整的 frontmatter：

```yaml
---
title: 学科名称
aliases:
  - 英文名称
  - 其他别名
category:
  - 金融经济百科
tags:
  - finance/economics/politics/law
  - cn-us-comparison
type: topic/appendix
status: stable/draft
importance: core/important/normal
version: vX.X
date: 'YYYY-MM-DD'
---
```

**字段说明**：
- `title`: 文档标题（中文）
- `aliases`: 别名列表（英文、其他称呼）
- `category`: 所属分类
- `tags`: 标签（学科 + 特性）
- `type`: 文档类型（topic=主题, appendix=附录）
- `status`: 状态（stable=稳定, draft=草稿）
- `importance`: 重要性（core=核心, important=重要, normal=普通）
- `version`: 版本号（v1.0, v1.1 等）
- `date`: 最后更新日期（YYYY-MM-DD）

### 知识点结构（五段式）

每个核心知识点必须采用统一的五段式结构：

#### 1. 理论定义
- 概念的学术定义
- 核心特征与功能
- 理论背景与发展

#### 2. 数学化表述
- 公式表达（使用 LaTeX）
- 变量说明
- 数学关系

示例：
```markdown
$$P_0 = \sum_{t=1}^{\infty} \frac{D_t}{(1+r)^t}$$

其中 $D_t$ 为第 $t$ 年股利，$r$ 为必要收益率。
```

#### 3. 中美对比表格
- 必须包含"维度、中国、美国、对比要点"四列
- 关键差异用引用块强调

示例：
```markdown
| 维度 | 中国 | 美国 | 对比要点 |
|------|------|------|----------|
| 监管体系 | "一委一行两会" | Fed+SEC+CFTC | 中国监管更集中 |

> **中美关键差异**：中国监管体系集中化程度更高...
```

#### 4. 直觉理解
- 用通俗语言解释概念本质
- 类比、比喻、生活化例子
- 避免术语堆砌

#### 5. 实践指南
- 投资者关注点
- 分析要点
- 操作建议
- 风险提示

### Obsidian 链接语法

## 版本管理规范

### 版本号格式

采用三段式版本号：`v[major].[minor].[patch]`

| 版本类型 | 触发条件 | 示例 |
|---------|---------|------|
| Major | 架构重组、学科增删、重大内容调整 | v1.0 → v2.0 |
| Minor | 知识点新增、章节扩充、内容补充 | v1.0 → v1.1 |
| Patch | 格式修正、错别字修复、链接修正 | v1.0 → v1.0.1 |

### 版本存档流程

**重要**：Major 和 Minor 版本更新前必须存档旧版本。

1. **存档旧版本**：
   ```bash
   cp 02-finance.md archive/02-finance-v1.0-20260419.md
   ```

2. **更新文档内容**

3. **更新 frontmatter**：
   ```yaml
   version: v1.1
   date: '2026-04-20'
   ```

4. **添加变更说明**（可选，在文档末尾）：
   ```markdown
   ## 版本历史
   
   - v1.1 (2026-04-20): 新增衍生品章节
   - v1.0 (2026-04-19): 初始版本
   ```

## Git 工作流程

### 分支策略

- `main`: 稳定版本（status: stable）
- `draft`: 草稿内容（status: draft）
- `feature/*`: 新增学科或大型功能

### 提交规范

采用 Conventional Commits 格式：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**：
- `feat`: 新增知识点或章节
- `fix`: 修正错误内容
- `docs`: 文档更新（README、CLAUDE.md）
- `style`: 格式调整（不影响内容）
- `refactor`: 结构重组
- `chore`: 其他杂项

**示例**：
```
feat(finance): 新增期权策略章节

- 添加期权希腊字母风险度量
- 补充期权策略矩阵
- 更新版本号至 v1.1

Closes #12
```

### 提交前检查

```bash
# 1. 检查文件状态
git status

# 2. 查看变更内容
git diff

# 3. 添加文件
git add 02-finance.md

# 4. 提交（附带详细说明）
git commit -m "feat(finance): 新增期权策略章节"

# 5. 推送（如有远程仓库）
git push origin main
```

## 协作规范

### 新增知识点流程

1. **创建分支**：
   ```bash
   git checkout -b feature/add-derivatives
   ```

2. **编写内容**：
   - 遵循五段式结构
   - 包含中美对比
   - 添加相关链接

3. **自查质量**：
   - 运行质量检查清单
   - 确认格式规范

4. **提交变更**：
   ```bash
   git add .
   git commit -m "feat(finance): 新增衍生品章节"
   ```

5. **合并到主分支**：
   ```bash
   git checkout main
   git merge feature/add-derivatives
   ```

### 版本更新流程

1. **存档旧版本**（Major/Minor 更新）
2. **更新内容**
3. **更新 frontmatter**（version + date）
4. **提交变更**
5. **打标签**（可选）：
   ```bash
   git tag -a v1.1 -m "新增衍生品章节"
   git push origin v1.1
   ```

## 常见问题

### Q1: 如何判断是否需要存档？

**A**: Major 和 Minor 版本更新需要存档，Patch 更新不需要。

### Q2: 数学公式如何编写？

**A**: 使用 LaTeX 语法，行内公式用 `$...$`，独立公式用 `$$...$$`。

### Q3: 如何处理跨学科内容？

**A**: 放在 `05-cross-themes.md` 中，并在相关学科文档中添加链接。

### Q4: 如何添加新的学科？

**A**: 
1. 创建新文件（如 `06-sociology.md`）
2. 更新 `00-index.md` 导航
3. 更新 `README.md` 结构说明
4. 在相关文档中添加链接

### Q5: 如何处理 draft 状态文档？

**A**: 
- draft 文档可以不完整
- 完成后更新 `status: stable`
- 更新版本号和日期
